## üöÄ Quick Start

### Prerequisites

- Node.js 18+ 
- npm or pnpm
- A [Convex](https://convex.dev) account (free tier available)
- A [Polar.sh](https://polar.sh) account for payments

### Installation

```bash
# 1. Clone the repository
git clone <your-repo-url>
cd <project-folder>

# 2. Install dependencies
npm install

# 3. Copy the environment example file
cp .env.example .env.local

# 4. Set up Convex
npx convex dev

# This will:
# - Create a new Convex project (or link to existing)
# - Generate your CONVEX_DEPLOYMENT and VITE_CONVEX_URL
# - Start the Convex dev server

# 5. Start the development server (in a new terminal)
npm run dev
```

The app will be available at `http://localhost:5173`

## üîß Environment Variables

Create a `.env.local` file in the root directory with the following variables:

### Required Variables

```env
# Convex (auto-generated by `npx convex dev`)
CONVEX_DEPLOYMENT=dev:your-deployment-name
VITE_CONVEX_URL=https://your-deployment.convex.cloud

# Site URL
SITE_URL=http://localhost:5173
```

### Authentication (OAuth)

**Google OAuth Setup:**
1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select existing
3. Navigate to **APIs & Services** ‚Üí **Credentials**
4. Create **OAuth 2.0 Client ID** (Web application)
5. Add authorized redirect URI: `https://your-convex-deployment.convex.site/api/auth/callback/google`
6. Copy the Client ID and Client Secret

**GitHub OAuth Setup:**
1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
2. Create a new **OAuth App**
3. Set Authorization callback URL: `https://your-convex-deployment.convex.site/api/auth/callback/github`
4. Copy the Client ID and Client Secret

**JWT Keys:** Convex Auth requires JWT keys. Generate them with:
```bash
openssl genpkey -algorithm RSA -out private_key.pem -pkeyopt rsa_keygen_bits:2048
openssl rsa -pubout -in private_key.pem -out public_key.pem
```

```env
# Google OAuth
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

# GitHub OAuth
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret

# JWT for Convex Auth
JWT_PRIVATE_KEY=your_jwt_private_key

```

## üîê Authentication

Authentication is handled via **Convex Auth** with Google and GitHub OAuth providers.

### How It Works

1. Users click "Sign in with Google" or "Sign in with GitHub"
2. They're redirected to the OAuth provider
3. After approval, they're redirected back to your Convex deployment
4. Convex creates/updates the user and establishes a session
5. A welcome email is automatically sent to new users

### Callback URLs

When setting up OAuth apps, use these callback URLs:
- **Google:** `https://your-convex-deployment.convex.site/api/auth/callback/google`
- **GitHub:** `https://your-convex-deployment.convex.site/api/auth/callback/github`

> üí° Replace `your-convex-deployment` with your actual Convex deployment name (found in your Convex dashboard).

See [Environment Variables](#-environment-variables) for full setup instructions.

## üìä Analytics Setup

### PostHog

[PostHog](https://posthog.com) is included for product analytics, feature flags, session recordings, and A/B testing.

**Setup:**
1. Create account at [posthog.com](https://posthog.com)
2. Create a new project
3. Copy your Project API Key
4. Add to `.env.local`:
   ```env
   VITE_POSTHOG_KEY=phc_your_key
   VITE_POSTHOG_HOST=https://eu.i.posthog.com  # or https://us.i.posthog.com
   ```

**Features Enabled by Default:**
- Autocapture (clicks, form submissions, page views)
- Pageview tracking on navigation
- Session recording
- Do Not Track browser setting respected

**Using the Analytics Hook:**

```tsx
import { usePostHogAnalytics } from '@/hooks/usePostHogAnalytics'

function MyComponent() {
  const { capture, setPersonProperties } = usePostHogAnalytics()

  const handlePurchase = () => {
    capture('purchase_completed', {
      product_id: 'prod_123',
      amount: 99.99,
      currency: 'USD'
    })
  }

  return <button onClick={handlePurchase}>Buy Now</button>
}
```

**Available Methods:**

| Method | Description |
|--------|-------------|
| `capture(event, properties?)` | Track a custom event |
| `identify(userId, properties?)` | Manually identify a user |
| `reset()` | Reset user identity (auto-done on logout) |
| `setPersonProperties(properties)` | Set persistent user properties |
| `isFeatureEnabled(flagKey)` | Check if a feature flag is enabled |
| `getFeatureFlag(flagKey)` | Get feature flag variant |
| `optOut()` / `optIn()` | Manage tracking consent |

**Events Tracked Automatically:**

| Event | Location |
|-------|----------|
| `sign_in_clicked` | Header, Auth |
| `sign_out_clicked` | Header |
| `theme_toggled` | Header |
| `checkout_started` | Pricing page |
| `credits_used` | Home page demo |
| `account_deleted` | Settings |

**Feature Flags:**

```tsx
import { usePostHogAnalytics } from '@/hooks/usePostHogAnalytics'

function MyComponent() {
  const { isFeatureEnabled } = usePostHogAnalytics()

  if (isFeatureEnabled('beta-feature')) {
    return <BetaFeature />
  }
  return <StandardFeature />
}
```

**Privacy - Opt Out:**

```tsx
const { optOut, optIn, hasOptedOut } = usePostHogAnalytics()

// Allow users to opt out
<button onClick={() => optOut()}>Disable Tracking</button>
```

## üìù Blog System

The template includes a simple, file-based blog powered by Markdown. No database required‚Äîjust add `.md` files and they're automatically discovered.

### How It Works

1. Blog posts are stored as Markdown files in `src/content/blog/`
2. Each file uses YAML frontmatter for metadata
3. Vite's glob imports automatically discover all posts at build time
4. Posts are sorted by date (newest first)

### Creating a New Post

1. Create a new `.md` file in `src/content/blog/`:

```markdown
---
title: "Your Post Title"
slug: "your-post-slug"
date: "2025-12-15"
description: "A brief description for SEO and previews."
author: "Your Name"
---

# Your Post Title

Your content here. Full **Markdown** support including:

- Lists
- **Bold** and *italic* text
- [Links](https://example.com)
- Code blocks
- Images

```

2. That's it! The post will appear at `/blog/your-post-slug`

### Frontmatter Fields

| Field | Required | Description |
|-------|----------|-------------|
| `title` | Yes | Post title displayed on the page |
| `slug` | No | URL slug (defaults to filename if omitted) |
| `date` | Yes | Publication date (YYYY-MM-DD format) |
| `description` | Yes | Short description for SEO and blog listing |
| `author` | No | Author name |

### Using Blog Data in Code

```tsx
import { getAllPosts, getPostBySlug } from '@/lib/blog';

// Get all posts (sorted by date)
const posts = await getAllPosts();

// Get a specific post
const post = await getPostBySlug('your-post-slug');
```

### Available Routes

| Route | Description |
|-------|-------------|
| `/blog` | Blog listing page with all posts |
| `/blog/:slug` | Individual blog post page |

### Updating the Sitemap

After adding new blog posts, regenerate the sitemap to include them:

```bash
npm run sitemap
```

> üí° This runs automatically during `npm run build` via the `postbuild` script.
